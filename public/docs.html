API Endpoints

  • POST /api/exchange — Exchange OAuth code for tokens
  • POST /api/refresh — Refresh access token
  • GET /api/reviews?pkg=[PACKAGE] — Fetch Play Store reviews
  • GET /api/live — Health check; returns {"status":"live"}

cURL Examples

Exchange Auth Code

curl -X POST https://<your-domain>/api/exchange \
  -H "Content-Type: application/json" \
  -d '{
    "code": "AUTH_CODE_HERE",
    "client_id": "YOUR_CLIENT_ID",
    "client_secret": "YOUR_CLIENT_SECRET"
  }'

Refresh Token

curl -X POST https://<your-domain>/api/refresh \
  -H "Content-Type: application/json" \
  -d '{
    "refresh_token": "REFRESH_TOKEN_HERE",
    "client_id": "YOUR_CLIENT_ID",
    "client_secret": "YOUR_CLIENT_SECRET"
  }'

Fetch Reviews

curl "https://<your-domain>/api/reviews?pkg=com.example.app" \
  -H "Authorization: Bearer ACCESS_TOKEN_HERE"

Health Check

curl https://<your-domain>/api/live

Google Cloud Setup

  1. Go to the Google Cloud Console and select or create a project.
  2. Enable the Android Publisher API under APIs & Services > Library.
  3. Navigate to APIs & Services > Credentials.
    • Create an OAuth client ID of type Web application.
    • Under Authorized redirect URIs, add postmessage.
    • Copy the generated Client ID and Client Secret.

index.html UI Structure

  • Header: App title banner.
  • Config Section (#config): Inputs for Client ID/Secret and Initialize button.
  • Auth Section (#auth):
    • Sign In button — triggers OAuth code flow.
    • Refresh Token button — calls /api/refresh.
    • #tokens area — displays current tokens.
  • App Section (#app):
    • Input for Play Store package name.
    • Fetch Reviews button — calls /api/reviews.
    • #reviews grid — displays review cards.
  • Logger (#log): Shows all actions and raw API responses for debugging.
